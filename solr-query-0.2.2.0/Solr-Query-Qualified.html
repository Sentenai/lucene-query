Description

This module is an alternative to Solr.Query that does not export any operators, and is intended to be imported qualified, because it contains function names that clash with the Prelude.

import qualified Solr.Query.Qualified as Solr

Here is a quick conversion guide:

(~:)  = fuzz
(^:)  = boost
(=:)  = field
(&&:) = and
(||:) = or
(-:)  = not
(^=:) = score

Query construction

For simplicity, the type signatures in the examples below monomorphise the functions to use SolrQuery (and therefore SolrExpr, due to the functional dependency).

defaultField :: Solr expr query => expr a -> query Source

A default field query.

Example:

@ -- bar query :: Solr.SolrQuery query = Solr.word "bar"

field :: Solr expr query => Text -> expr a -> query Source

A field query.

Example:

-- foo:bar
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.word "bar")

and :: Solr expr query => query -> query -> query infixr 3 Source

An AND query.

Example:

-- foo:bar AND baz:qux
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.word "bar") `Solr.and` Solr.field "baz" (Solr.word "qux")

or :: Solr expr query => query -> query -> query infixr 2 Source

An OR query.

Example:

-- foo:bar OR baz:qux
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.word "bar") `Solr.or` Solr.field "baz" (Solr.word "qux")

not :: Solr expr query => query -> query -> query infixr 1 Source

A NOT, '!' or '-' query.

Example:

-- foo:bar NOT baz:qux
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.word "bar") `Solr.not` Solr.field "baz" (Solr.word "qux")

score :: Solr expr query => query -> Float -> query infixr 4 Source

The '^=' constant score operator.

This is given right-fixity to reject queries like q ^= 1 ^= 2, which may very well be a valid Solr query (I haven't tested), but are nonetheless nonsense.

Example:

-- (foo:bar)^=3.5
query :: Solr.SolrQuery
query = Solr.score (Solr.field "foo" (Solr.word "bar")) 3.5

Expression construction

For simplicity, the type signatures in the examples below monomorphise the functions to use SolrQuery (and therefore SolrExpr, due to the functional dependency).

int :: Solr expr query => Int -> expr TInt Source

An int expression.

Note that sometimes you may use the Num instance for SolrExpr TInt, but usually an explicit type signature will be required (at the interpretation site or earlier).

Example:

-- foo:5
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.int 5)

A false expression.

Example:

-- foo:false

word :: Solr expr query => Text -> expr TWord Source

A single word. Must not contain any spaces, wildcard characters ('?' and '*'), or tildes ('~'), though this is not enforced by the type system.

Note that sometimes you may use the IsString instance for SolrExpr TWord, but usually an explicit type signature will be required (at the interpretation site or earlier).

Example:

-- foo:bar
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.word "bar")

-- foo:bar
query :: Solr.SolrQuery
query = Solr.field "foo" ("bar" :: Solr.SolrExpr TWord)

wild :: Solr expr query => Text -> expr TWild Source

A single word that may contain wildcard characters ('?' and '*'), although the meaning of consecutive '*'s is probably ill-defined. Must also not contain any spaces or tildes ('~'), though this is not enforced by the type system.

Example:

-- foo:b?r
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.wild "b?r")

regex :: Solr expr query => Text -> expr TRegex Source

A regular expression, whose syntax is described by http://lucene.apache.org/core/5_5_0/core/org/apache/lucene/util/automaton/RegExp.html?is-external=true.

Note that the leading and trailing '/' must be omitted. The regex innards are not type checked in any way.

-- foo:/[mb]oat/
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.regex "[mb]oat")

phrase :: Solr expr query => [expr TWord] -> expr TPhrase Source

A phrase, composed of multiple (non-fuzzy) words, none of which may contain wildcard characters. Both of these properties are enforced by the type system, as long as the words themselves adhere to the word contract. The list should not be empty.

Note that sometimes you may use the IsList instance for SolrExpr TPhrase, but usually an explicit type signature will be required (at the interpretation site or earlier).

Example:

-- foo:"bar baz"
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.phrase ["bar", "baz"]) -- ok

-- foo:"bar b?z" (an invalid Solr query)
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.phrase ["bar", Solr.wild "b?z"]) -- type error

-- foo:"bar b?z" (an invalid Solr query)
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.phrase ["bar", "b?z"]) -- breaks word contract

Or, with OverloadedLists:

-- foo:"bar baz"
query :: Solr.SolrQuery
query = Solr.field "foo" (["bar", "baz"] :: Solr.SolrExpr TPhrase)

fuzz :: (Solr expr query, FuzzableType a) => expr a -> Int -> expr (TFuzzed a) Source

The '~' operator, which fuzzes its argument (either a word or phrase) by a numeric amount.

This will have one of the following two types:

fuzz :: SolrExpr TWord   -> Int -> SolrExpr TFuzzyWord   -- Int must be 0, 1, or 2
fuzz :: SolrExpr TPhrase -> Int -> SolrExpr TFuzzyPhrase -- Int must be positive

Example:

-- foo:bar~1
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.fuzz (Solr.word "bar") 1)

-- foo:"bar baz qux"~10
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.fuzz (Solr.phrase ["bar", "baz", "qux"]) 10)

fuzzy :: Solr expr query => expr TWord -> expr TFuzzyWord Source

Short-hand for fuzzing a word by 2. This is the default behavior of a Solr '~' operator without an integer added.

fuzzy e = fuzz e 2

Example:

-- foo:bar~
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.fuzzy "bar")

to :: (Solr expr query, PrimType a) => Boundary (expr a) -> Boundary (expr a) -> expr TRange infix 6 Source

A range expression.

This will have one of the following two types:

to :: Boundary (SolrExpr TWord) -> Boundary (SolrExpr TWord) -> SolrExpr TRange
to :: Boundary (SolrExpr TInt)  -> Boundary (SolrExpr TInt)  -> SolrExpr TRange

Example:

-- foo:[5 TO 10}
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.incl (Solr.int 5) `Solr.to` Solr.excl (Solr.int 10))

gt :: (Solr expr query, PrimType a) => expr a -> expr TRange Source

Short-hand for a greater-than range query.

gt e = excl e `to` star

Example:

-- foo:>5
-- foo:{5 TO *]
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.gt (Solr.int 5))

gte :: (Solr expr query, PrimType a) => expr a -> expr TRange Source

Short-hand for a greater-than-or-equal-to range query.

gte e = incl e `to` star

Example:

-- foo:>=5
-- foo:[5 TO *]
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.gte (Solr.int 5))

lt :: (Solr expr query, PrimType a) => expr a -> expr TRange Source

Short-hand for a less-than range query.

 lt e = star `to` excl e

Example:

-- foo:<5
-- foo:[* TO 5}
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.lt (Solr.int 5))

lte :: (Solr expr query, PrimType a) => expr a -> expr TRange Source

Short-hand for a less-than-or-equal-to range query.

lte e = star `to` incl e

Example:

-- foo:<=5
-- foo:[* TO 5]
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.lte (Solr.int 5))

boost :: (Solr expr query, BoostableType a) => expr a -> Float -> expr (TBoosted a) Source

The '^' operator, which boosts its argument.

This will have one of the following two types:

boost :: SolrExpr TWord   -> Float -> SolrExpr TBoostedWord
boost :: SolrExpr TPhrase -> Float -> SolrExpr TBoostedPhrase

Example:

-- foo:bar^3.5
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.boost (Solr.word "bar") 3.5)

-- foo:"bar baz"^3.5
query :: Solr.SolrQuery
query = Solr.field "foo" (Solr.boost (Solr.phrase ["bar", "baz"]) 3.5)
