Description

This module is an alternative to Solr.Class that does not export any operators, and is intended to be imported qualified, because it contains function names that clash with the Prelude.

import qualified Solr.Qualified.Class as Solr

Here is a quick conversion guide:

(~:)  = fuzz
(^:)  = boost
(=:)  = field
(&&:) = and
(||:) = or
(-:)  = not
(^=:) = score

An int expression.

Note that sometimes you may use the Num instance for SolrExpr TInt, but usually an explicit type signature will be required (at the interpretation site or earlier).

query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.int 5)

word :: SolrExprSYM expr => Text -> expr TWord Source

A single word. Must not contain any spaces, wildcard characters ('?' and '*'), or tildes ('~'), though this is not enforced by the type system.

Note that sometimes you may use the IsString instance for SolrExpr TWord, but usually an explicit type signature will be required (at the interpretation site or earlier).

Example:

-- foo:bar
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.word "bar")

-- foo:bar
query :: Solr.SolrQuery 'False
query = Solr.field "foo" ("bar" :: Solr.SolrExpr TWord)

wild :: SolrExprSYM expr => Text -> expr TWild Source

A single word that may contain wildcard characters ('?' and '*'), although the meaning of consecutive '*'s is probably ill-defined. Must also not contain any spaces or tildes ('~'), though this is not enforced by the type system.

Example:

-- foo:b?r
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.wild "b?r")

regex :: SolrExprSYM expr => Text -> expr TRegex Source

A regular expression, whose syntax is described by http://lucene.apache.org/core/5_5_0/core/org/apache/lucene/util/automaton/RegExp.html?is-external=true.

Note that the leading and trailing '/' must be omitted. The regex innards are not type checked in any way.

-- foo:/[mb]oat/
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.regex "[mb]oat")

phrase :: SolrExprSYM expr => [expr TWord] -> expr TPhrase Source

A phrase, composed of multiple (non-fuzzy) words, none of which may contain wildcard characters. Both of these properties are enforced by the type system, as long as the words themselves adhere to the word contract. The list should not be empty.

Note that sometimes you may use the IsList instance for SolrExpr TPhrase, but usually an explicit type signature will be required (at the interpretation site or earlier).

Example:

-- foo:"bar baz"
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.phrase ["bar", "baz"]) -- ok

-- foo:"bar b?z" (an invalid Solr query)
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.phrase ["bar", Solr.wild "b?z"]) -- type error

-- foo:"bar b?z" (an invalid Solr query)
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.phrase ["bar", "b?z"]) -- breaks word contract

Or, with OverloadedLists:

-- foo:"bar baz"
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (["bar", "baz"] :: Solr.SolrExpr TPhrase)

fuzz :: (SolrExprSYM expr, FuzzableType a) => expr a -> Int -> expr (TFuzzed a) Source

The '~' operator, which fuzzes its argument (either a word or phrase) by a numeric amount.

This will have one of the following two types:

fuzz :: SolrExpr TWord   -> Int -> SolrExpr TFuzzyWord   -- Int must be 0, 1, or 2
fuzz :: SolrExpr TPhrase -> Int -> SolrExpr TFuzzyPhrase -- Int must be positive

Example:

-- foo:bar~1
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.fuzz (Solr.word "bar") 1)

-- foo:"bar baz qux"~10
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.fuzz (Solr.phrase ["bar", "baz", "qux"]) 10)

to :: (SolrExprSYM expr, PrimType a) => Boundary (expr a) -> Boundary (expr a) -> expr TRange infix 6 Source

A range expression.

This will have one of the following two types:

to :: Boundary (SolrExpr TWord) -> Boundary (SolrExpr TWord) -> SolrExpr TRange
to :: Boundary (SolrExpr TInt)  -> Boundary (SolrExpr TInt)  -> SolrExpr TRange

Example:

-- foo:[5 TO 10}
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.incl (Solr.int 5) `Solr.to` Solr.excl (Solr.int 10))

boost :: (SolrExprSYM expr, BoostableType a) => expr a -> Float -> expr (TBoosted a) Source

The '^' operator, which boosts its argument.

This will have one of the following two types:

boost :: SolrExpr TWord   -> Float -> SolrExpr TBoostedWord
boost :: SolrExpr TPhrase -> Float -> SolrExpr TBoostedPhrase

Example:

-- foo:bar^3.5
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.boost (Solr.word "bar") 3.5)

-- foo:"bar baz"^3.5
query :: Solr.SolrQuery 'False
query = Solr.field "foo" (Solr.boost (Solr.phrase ["bar", "baz"]) 3.5)

score :: SolrQuerySYM expr query => query False -> Float -> query False infixr 4 Source

The '^=' constant score operator.

This is given right-fixity to reject queries like q ^= 1 ^= 2, which may very well be a valid Solr query (I haven't tested), but are nonetheless nonsense.

Example:

-- (foo:bar)^=3.5
query :: Solr.SolrQuery 'False
query = Solr.score (Solr.field "foo" (Solr.word "bar")) 3.5

localParams :: SolrQuerySYM expr query => LocalParams query -> query False -> query True Source

Add local parameters to a query. A query can only have one set of local parameters, hence the boolean tag that tracks whether or not they've been added.

Example:

-- {!df=foo}bar
query :: Solr.SolrQuery 'True
query = Solr.localParams (Solr.paramDefaultField "foo") (Solr.defaultField (Solr.word "bar"))
